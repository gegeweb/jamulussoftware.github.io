Instalación en Linux

Asegúrate de leer la página de Cómo Empezar.

Ahora mismo no ofrecemos paquetes oficiales para Jamulus

Hay una lista no oficial de paquetes para distintas distribuciones aquí. También puedes utilizar un script todo-en-uno si lo deseas.

Mientras tanto, compilar las fuentes es bastante fácil:

Obtén las fuentes de Jamulus

  1. Abre una terminal (línea de comandos - CTRL+ALT+t en Ubuntu y distros relacionadas)
  2. Descarga y descomprime las fuentes para la última versión: 
    wget https://github.com/corrados/jamulus/archive/latest.tar.gz
tar -xvf latest.tar.gz

Instala las dependencias

Primero actualiza la lista de paquetes (por ej. en distribuciones basadas en Debian con sudo apt-get update).

En distros basadas en Ubuntu 18.04+, Debian 9+ o 10 y Raspberry Pi Raspbian Buster o superior:

sudo apt-get install build-essential qt5-qmake qtdeclarative5-dev qt5-default qttools5-dev-tools libjack-jackd2-dev

En Fedora:

sudo dnf install qt5-qtdeclarative-devel jack-audio-connection-kit-dbus jack-audio-connection-kit-devel

Qjackctl: Opcional, pero recomendado

QjackCtl es una utilidad para ayudarte a configurar Jack (instalado como parte de las dependencias mencionadas arriba). Instálalo por ej. con

sudo apt-get install qjackctl

También puedes plantearte usar un kernel low-latency (por ej. para Ubuntu 18.04: sudo apt-get install linux-lowlatency-hwe-18.04).

Compílalo

Ahora cd a la carpeta de las fuentes de Jamulus que has descargado:

cd jamulus-latest

Ahora compila las fuentes con los siguientes comandos (el último make puede tardar varios minutos):

qmake Jamulus.pro
make clean
make
sudo make install

Configura tu tarjeta de sonido

Configura Jack con QJackCtl

Los clientes de Jamulus necesitan Jack para funcionar, pero hay que configurarlo primero. El método recomendado es utilizar QjackCtl..

  1. Abre la terminal por ej. con Crtl-Alt-T y
  2. Ejecuta el comando qjackctl. Verás la aplicación Jack Audio Connection Kit
  3. Configura tu interfaz de audio como sigue (la configuración exacta de Jack dependerá de lo que es capaz tu interfaz/tarjeta de audio):
  • Elige el Interfaz que quieres utilizar (puede haber varios en la lista)
  • Establece la Frecuencia de Muestreo en 48000
  • Establece los Cuadros/Periodo en 128 y Periodos/Buffer en 2 para empezar

Reinicia Jack para que surta efecto la nueva configuración

Arranca Jamulus

  1. Abre la terminal por ej. con Crtl-Alt-T y
  2. Ejecuta el comando Jamulus (con ‘J’ mayúscula) para iniciar el Cliente de Jamulus

Jamulus se instala en /usr/local/bin. Ahora puedes eliminar la carpeta de las fuentes si quieres.

Si tienes problemas con cortes de sonido (en particular errores de XRUN mostrados por Jack/QJackctl) prueba con valores más altos (por ej. 256 cuadros o 3 periodos) en el paso 3 anterior. Valores más bajos (por ej. 64 cuadros) pueden ofrecer un mejor rendimiento pero también quizá más problemas con el audio. Ver la página de resolución de problemas.

¿Todo instalado?

Mírate la Configuración de Hardware. Para información detallada sobre la utilización de Jamulus, por favor consulta el Manual de Ayuda.

Para actualizar tu instalación a una nueva versión

Descarga las nuevas fuentes como en Obtén Jamulus y repite solamente las instrucciones de compilación como si fuera una nueva instalación.

Nota para frikis

  • “make clean” es esencial para eliminar los archivos de Qt generados automáticamente los cuales están presentes en el archivo .tar.gz y que quizá no concuerden con la versión de Qt que estás utilizando.

  • Para utilizar una librería compartida de OPUS en lugar del que viene integrado usa qmake "CONFIG+=opus_shared_lib" Jamulus.pro.

  • Para utilizar este archivo configura el software con qmake "CONFIG+=noupcasename" Jamulus.pro para asegurarte de que el nombre de destino de este software es jamulus en lugar de Jamulus.

  • Usuarios de Raspberry Pi: Quizá quieras compilar el cliente en otra máquina y ejecutar el archivo binario en el Raspberry Pi. En este caso, las únicas librerías que necesitas para ejecutarlo son aquellos necesarios para la compilación de un servidor “headless” pero con las librerías de audio para Jack. Sobre todo lee el pie de página sobre la compilación “headless”.

  • A partir de la versión 3.5.3, Jamulus ya no es compatible con Qt4.
